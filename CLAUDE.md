## Project Overview

Google Fonts Tools (`gftools`) is a Python CLI toolkit with 88+ subcommands for building, fixing, validating, and managing fonts for the Google Fonts collection. The main entry point is `gftools <subcommand>`.

## Development Commands

### Installation
```bash
pip install -e '.[qa,test]'       # Full install with QA and test extras
pip install -e '.[test]'          # Without QA (needed on Python 3.14 where skia-python is unavailable)
```
System dependencies (Ubuntu): `libcairo2-dev`, `ffmpeg`

### Testing
```bash
pytest tests/*.py                 # Main test suite
pytest tests/push/*.py            # Push workflow tests (requires GF API secrets)
pytest tests/test_builder.py      # Builder tests only
pytest tests/test_builder.py::test_glyphs -v  # Single test
```

### Linting
```bash
black . --check --diff --color    # Check formatting
black .                           # Auto-format
```
Black config: targets Python 3.10, excludes `*_pb2.py` (autogenerated protobuf files).

## Architecture

### Package Layout
Source code lives under `Lib/gftools/` (configured via `tool.setuptools.packages.find` with `where = ["Lib"]`).

### CLI System (`Lib/gftools/scripts/`)
Each `scripts/*.py` module is a subcommand. The `scripts/__init__.py` auto-discovers them via glob and dynamically imports the requested module. Subcommands are also registered as individual entry points (`gftools-<name>`) in `pyproject.toml`.

### Builder System (`Lib/gftools/builder/`)
The central component for compiling font projects from source (Glyphs, UFO, Designspace files):
- **`__init__.py`** — `GFBuilder` class: parses YAML config, validates against strictyaml schema, builds a networkx dependency graph, generates a Ninja build file, then runs Ninja for parallel compilation
- **`schema.py`** — YAML config validation schemas (`BASE_SCHEMA`, `GOOGLEFONTS_SCHEMA`)
- **`recipeproviders/`** — Generate build recipes automatically (`googlefonts` default, `noto`)
- **`operations/`** — 30+ build operation types (compile, fix, subset, hint, etc.) with `OperationBase` + `OperationRegistry`
- **`jobrunner/`** — Execution backend for build jobs

### Key Modules
- **`fix.py`** — Font fix functions for Google Fonts spec compliance
- **`utils.py`** — General utilities shared across scripts
- **`packager/`** — Font family packaging for Google Fonts PRs
- **`push/`** — Deployment/push workflow management
- **`subsetmerger.py`** — Subset font merging
- **`qa.py`** — Quality assurance with fontbakery integration
- **`*.proto` / `*_pb2.py`** — Protobuf definitions for font metadata (METADATA.pb)

### Versioning
Dynamic versioning via `setuptools_scm` from git tags. Auto-generates `Lib/gftools/_version.py`. Releases are triggered by pushing annotated `v*` tags, which build and publish to PyPI via GitHub Actions.

## Key Conventions
- Python 3.10+ required
- Protobuf pinned to v3 (`>=3.19.4, <4`) — `*_pb2.py` files are pre-compiled, do not regenerate without matching protoc version
- Test fixtures live in `data/test/`
- Builder YAML configs validated with strictyaml then loaded with PyYAML for ease of use
